# <a name="resource-qualifiers-and-visualization-options"></a>Квалификаторы ресурсов и параметры визуализации

_В этом разделе объясняется, как определить ресурсы, которые будут использоваться только при совпадении некоторых значений квалификаторов. Простой пример — это строковый ресурс с указанием языка. Строковый ресурс может быть определен по умолчанию, а другие альтернативные ресурсы, определенные для использования на дополнительных языках. Все типы ресурсов можно уточнять, включая сам макет._

# <a name="visual-studiotabwindows"></a>[Visual Studio](#tab/windows)

## <a name="resource-qualifier-options"></a>Параметры квалификатора ресурса

Чтобы получить доступ к **параметрам квалификатора ресурса** , щелкните значок с многоточием справа от кнопки **альбомный** режим:

[![Параметры квалификатора ресурса](resource-qualifiers-images/vs/08-resource-qual-opt-sml.png)](resource-qualifiers-images/vs/08-resource-qual-opt.png#lightbox)

Это диалоговое окно представляет собой раскрывающееся меню для следующих квалификаторов ресурсов:

- **Языковая** &ndash; Отображает доступные языковые ресурсы и предлагает возможность добавления новых ресурсов языков и регионов.

- **Режим пользовательского интерфейса** Отображает списки режимов (например, стыковочный модуль **автомобиля** и **стационарные закрепления**), а также направления макета. &ndash;

Каждое из этих раскрывающихся меню открывает новые диалоговые окна, в которых можно выбрать и настроить квалификаторы ресурсов (как описано далее).

### <a name="language"></a>Язык

Раскрывающееся меню **Language (язык** ) содержит только те языки, для которых определены ресурсы (или **все языки**, используемые по умолчанию). Однако имеется также параметр **Добавить язык/регион...** , который позволяет добавить в список новый язык:

[![Добавить язык или регион](resource-qualifiers-images/vs/09-add-language-region-sml.png)](resource-qualifiers-images/vs/09-add-language-region.png#lightbox)

При нажатии кнопки **Добавить язык/регион...** откроется диалоговое окно **Выбор языка** , в котором будут отображаться раскрывающиеся списки доступных языков и регионов:

![Список языков](resource-qualifiers-images/vs/10-languages.png "Список языков")

В этом примере мы выбрали **fr (французский)** для **языка и (** Бельгия) для регионального диалекта французский. Обратите внимание, что поле **Region** является необязательным, так как многие языки могут быть указаны без учета конкретных регионов. При повторном открытии раскрывающегося меню **Language (язык** ) отображается только что добавленный ресурс язык/регион:

![Выбранный язык и регион](resource-qualifiers-images/vs/11-language-region-added.png "Выбранный язык и регион")

Обратите внимание, что при добавлении нового языка, но для него не создаются новые ресурсы, добавленный язык больше не будет отображаться при следующем открытии проекта.

### <a name="ui-mode"></a>Режим пользовательского интерфейса

При выборе раскрывающегося меню **режим пользовательского интерфейса** отображается список режимов, например **Обычная**, **закрепление автомобиля**, **док-стол**, **телевизор**, **устройство**и **Контрольные значения**:

[![Меню режима пользовательского интерфейса](resource-qualifiers-images/vs/12-ui-mode-sml.png)](resource-qualifiers-images/vs/12-ui-mode.png#lightbox)

Под этим списком следуют ночные и **ночные режимы,** **а затем**направления разметки **слева направо** и **справа налево** (сведения о параметрах **слева направо** и **справа** налево) см. в разделе [. LayoutDirection](xref:Android.Util.LayoutDirection)).
Последними элементами в диалоговом окне **Параметры квалификатора ресурсов** являются **скругленные экраны** (для использования с износом Android) или **нескругленные экраны**.
Дополнительные сведения о круглых и нескругленных экранах см. в разделе [LAYOUTS](https://developer.android.com/training/wearables/ui/layouts.html).
Дополнительные сведения о режимах пользовательского интерфейса Android см. в разделе [уимодеманажер](xref:Android.App.UiModeManager).

## <a name="action-bar-settings"></a>Параметры панели действий

Значок **настройки панели действий** доступен слева от значка кисть (редактор темы):

![Параметры панели действий](resource-qualifiers-images/vs/14-action-bar.png "Параметры панели действий")

Этот значок открывает диалоговое окно контекстном меню Action, которое позволяет выбрать один из трех режимов панели действий:

- **Стандартный выпуск** &ndash; Состоит из эмблемы или значка и текста заголовка с необязательным подзаголовок.

- **Список** &ndash; Режим навигации по списку. Вместо статического текста заголовка этот режим представляет собой меню списка для навигации в рамках действия (то есть оно может быть представлено пользователю в виде раскрывающегося списка).

- **Вкладки** &ndash; Режим навигации по табуляции. В этом режиме вместо статического текста заголовка представлен ряд вкладок для навигации внутри действия.

## <a name="themes"></a>Темы

В раскрывающемся меню **Тема** отображаются все темы, определенные в проекте. При выборе пункта **Дополнительные темы** открывается диалоговое окно со списком всех тем, доступных в установленной пакет SDK для Android, как показано ниже:

[![Список дополнительных тем](resource-qualifiers-images/vs/15-theme-menu-sml.png "Список дополнительных тем")](resource-qualifiers-images/vs/15-theme-menu.png#lightbox)

При выборе темы область конструктора обновляется для отображения результата новой темы. Обратите внимание, что это изменение становится постоянным только при нажатии кнопки **ОК** в диалоговом окне **Тема** . После выбора темы она будет включена в раскрывающееся меню **Тема** , как показано ниже:

![Теперь доступна светло-тема](resource-qualifiers-images/vs/16-light-theme.png "Теперь доступна светло-тема")

## <a name="android-version"></a>Версия Android

Средство выбора **версии** Android задает версию Android, используемую для отрисовки макета в конструкторе. В селекторе отображаются все версии, совместимые с версией целевой платформы проекта:

![Список версий Android](resource-qualifiers-images/vs/17-android-version.png "Список версий Android")

Целевую версию .NET Framework можно задать в параметрах проекта в разделе **свойства > приложение > компилировать с использованием версии Android**. Дополнительные сведения о целевой версии .NET Framework см. в разделе Общие сведения об [уровнях API Android](~/android/app-fundamentals/android-api-levels.md).

Набор мини-приложений, доступных на панели элементов, определяется целевой версией платформы проекта. Это также справедливо для свойств, доступных в **окне Свойства**. Список доступных мини-приложений *не* определяется значением, выбранным в селекторе **версии** панели инструментов. Например, если вы настроили целевую версию проекта на Android 4,4, вы по-прежнему можете выбрать Android 6,0 на панели инструментов, чтобы увидеть, как выглядит проект в Android 6,0, но вы не сможете добавлять мини-приложения, относящиеся к Android 6,0 &ndash; .  по-прежнему будут ограничены только мини-приложения, доступные в Android 4,4.

Дополнительные сведения о типах ресурсов см. в разделе [ресурсы Android](~/android/app-fundamentals/resources-in-android/index.md).
