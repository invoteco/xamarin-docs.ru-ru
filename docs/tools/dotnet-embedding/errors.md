# <a name="net-embedding-errors"></a>Ошибки внедрения .NET

## <a name="em0xxx-binding-error-messages"></a>EM0xxx: привязка сообщений об ошибках

Например, параметры, среда

<!-- 0xxx: the generator itself, e.g. parameters, environment -->

<a name="EM0000" />

### <a name="em0000-unexpected-error---please-fill-a-bug-report-at-httpsgithubcommonoembeddinator-4000issues"></a>EM0000: непредвиденная ошибка. Заполните отчет об ошибке по адресу https://github.com/mono/Embeddinator-4000/issues

Произошло непредвиденное условие ошибки. Запишите [вопрос](https://github.com/mono/Embeddinator-4000/issues) с максимально возможной информацией, включая:

* Полные журналы сборки с максимальным уровнем детализации
* Минимальный тестовый случай, воспроизводящий ошибку
* Все сведения о версиях

Самый простой способ получить точную информацию о версии — использовать меню « **Xamarin Studio** », « **Xamarin Studio** элемент», « **отобразить сведения** » и скопировать/вставить сведения о версии (можно использовать кнопку « **Копировать информацию** »).

<a name="EM0001" />

### <a name="em0001-could-not-create-output-directory-x"></a>EM0001: не удалось создать выходной каталог `X`

Имя каталога, заданное `-o=DIR`, не существует и не может быть создано. Возможно, это недопустимое имя файловой системы.

<a name="EM0002" />

### <a name="em0002-option-x-is-not-supported"></a>EM0002: параметр `X` не поддерживается

Средство не поддерживает параметр `X`. Возможно, другая версия средства поддерживает его или не применяется в этой среде.

<a name="EM0003" />

### <a name="em0003-the-platform-x-is-not-valid"></a>EM0003: недопустимый `X` платформы.

Средство не поддерживает `X`платформы. Возможно, другая версия средства поддерживает его или не применяется в этой среде.

<a name="EM0004" />

### <a name="em0004-the-target-x-is-not-valid"></a>EM0004: недопустимый целевой `X`.

Средство не поддерживает целевой `X`. Возможно, другая версия средства поддерживает его или не применяется в этой среде.

<a name="EM0005" />

### <a name="em0005-the-compilation-target-x-is-not-valid"></a>EM0005: недопустимый целевой `X` компиляции.

Средство не поддерживает целевой `X`компиляции. Возможно, другая версия средства поддерживает его или не применяется в этой среде.

<a name="EM0006" />

### <a name="em0006-could-not-find-the-xcode-location"></a>EM0006: не удалось найти расположение Xcode.

Средству не удалось найти выбранное расположение Xcode с помощью команды `xcode-select -p`. Убедитесь, что эта команда выполнена, и возвращает правильное расположение Xcode.

<a name="EM0007" />

### <a name="em0007-could-not-get-the-sdk-version-for-sdk"></a>EM0007: не удалось получить версию пакета SDK для "{SDK}".

Средству не удалось получить версию пакета SDK с помощью команды `xcrun --show-sdk-version --sdk {sdk}`. Убедитесь, что эта команда выполнена, и возвращает версию пакета SDK.

<a name="EM0008" />

### <a name="em0008-the-architecture-arch-is-not-valid-for-platform-valid-architectures-for-platform-are-architectures"></a>EM0008: архитектура "{Arch}" недопустима для {Platform}. Допустимые архитектуры для {Platform}: "{Architectures}".

Архитектура в сообщении об ошибке недопустима для целевой платформы. Убедитесь, что параметру--ABI передана допустимая архитектура.

<a name="EM0009" />

### <a name="em0009-the-feature-x-is-not-currently-implemented-by-the-generator"></a>EM0009: функция `X` в настоящее время не реализована генератором

Это известная проблема, которую мы планируем исправить в будущих выпусках генератора. Публикации — это Добро пожаловать.

<a name="EM0010" />

### <a name="em0010-cant-merge-the-frameworks-simulatorframework-and-deviceframework-because-the-file-file-exists-in-both"></a>EM0010: не удается выполнить слияние платформ "{Симулаторфрамеворк}" и "{Девицефрамеворк}", так как файл "{File}" существует в обоих расположениях.

Средству не удалось выполнить слияние платформ, упомянутых в сообщении об ошибке, так как между ними есть общий файл.

Это может указывать на ошибку при внедрении .NET; Запишите отчет об ошибке в [https://github.com/mono/Embeddinator-4000/issues](https://github.com/mono/Embeddinator-4000/issues) с тестовым случаем.

<a name="EM0011" />

### <a name="em0011-the-assembly-x-does-not-exist"></a>EM0011: `X` сборки не существует.

Средству не удалось найти сборку `X`, указанную в аргументах.

<a name="EM0012" />

### <a name="em0012-the-assembly-name-x-is-not-unique"></a>EM0012: имя сборки `X` не является уникальным

У нескольких предоставленных сборок одно и то же внутреннее имя, и их невозможно будет отличить во время выполнения.

Наиболее вероятной причиной является то, что сборка указана в аргументах командной строки более одного раза. Однако переименованная сборка по-прежнему сохраняет ее исходное имя, и несколько копий не могут сосуществовать.

<a name="EM0013" />

### <a name="em0013-cant-find-the-assembly-x-referenced-by-y"></a>EM0013: не удается найти сборку "X", на которую ссылается "Y".

Средству не удалось найти сборку "X", на которую ссылается сборка "Y". Убедитесь, что все сборки, на которые имеются ссылки, находятся в одном каталоге с связываемой сборкой.

<a name="EM0014" />

### <a name="em0014-could-not-find-product-product-min_version-is-required"></a>EM0014: не удалось найти {Product} (требуется {Product} {min_version}).

Не удалось найти в системе зависимость, упомянутую в сообщении об ошибке.

<a name="EM0015" />

### <a name="em0015-could-not-find-a-valid-version-of-product-found-version-but-at-least-min_version-is-required"></a>EM0015: не удалось найти допустимую версию {Product} (обнаружено {Version}, но требуется по крайней мере {min_version}).

В системе обнаружена зависимость, упомянутая в сообщении об ошибке, но она слишком старая. Обновите версию до более новой.

<a name="EM0016" />

### <a name="em0016-could-not-create-symlink-file---target-error-number"></a>EM0016: не удалось создать символьную ссылку "{File}"-> "{Target}": ошибка {Number}

Не удалось создать символьную ссылку, упомянутый в сообщении об ошибке.

<a name="EM0026" />

### <a name="em0026-could-not-parse-the-command-line-argument-a-"></a>EM0026 не удалось проанализировать аргумент командной строки "A": *

Синтаксис, заданный для параметра командной строки, `A` не удалось проанализировать с помощью средства. Вероятно, это неверно. обратитесь к документации или в справке по правильному синтаксису.

<a name="EM0099" />

### <a name="em0099-internal-error--please-file-a-bug-report-with-a-test-case-httpsgithubcommonoembeddinator-4000issues"></a>EM0099: Внутренняя ошибка *. Запишите отчет об ошибке с тестовым случаем (https://github.com/mono/Embeddinator-4000/issues).

Это сообщение об ошибке сообщает о сбое внутренней проверки согласованности в внедрении .NET.

Это указывает на ошибку при внедрении .NET; Запишите отчет об ошибке в [https://github.com/mono/Embeddinator-4000/issues](https://github.com/mono/Embeddinator-4000/issues) с тестовым случаем.

## <a name="em1xxx-code-processing"></a>EM1xxx: обработка кода

<a name="EM1010" />

### <a name="em1010-type-t-is-not-generated-because-x-are-not-supported"></a>EM1010: тип `T` не создается, так как `X` не поддерживаются.

Это **предупреждение** , что тип `T` будет игнорироваться (т. е. не будет создаваться), так как в нем используется `X`, неподдерживаемая функция.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1011" />

### <a name="em1011-type-t-is-not-generated-because-it-lacks-marshaling-code-with-a-native-counterpart"></a>EM1011: тип `T` не создается, так как в нем отсутствует код упаковки с аналогом в машинном коде.

Это **предупреждение** , что тип `T` будет игнорироваться (т. е. не будет создаваться), так как он предоставляет что-то из .NET Framework, для которого требуется дополнительный маршалирование.

Примечание. это может поддерживаться с некоторыми ограничениями в будущей версии средства.

<a name="EM1020" />

### <a name="em1020-constructor-c-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1020: `C` конструктора не создается, так как тип параметра `T` не поддерживается.

Это **предупреждение** , что `C` конструктора будет игнорироваться (т. е. ничего не будет создано), так как параметр типа `T` не поддерживается.

Должно быть более раннее предупреждение, содержащее дополнительные сведения, почему тип `T` не поддерживается.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1021" />

### <a name="em1021-constructor-c-has-default-values-for-which-no-wrapper-is-generated"></a>EM1021: конструктор `C` имеет значения по умолчанию, для которых не создается оболочка.

Это **предупреждение о** том, что параметры по умолчанию конструктора `C` не создают никакого дополнительного кода. Наиболее распространенной причиной является то, что существующий метод уже имеет ту же сигнатуру. Например, в .NET может быть:

```csharp
public class MyType {
    public MyType () { ... }
    public MyType (int i = 0) { ... }
}
```

В таких случаях будут созданы только два созданных селектора `init`, оба из которых вызывают моно, но оболочка для более поздних версий не будет существовать.

<a name="EM1030" />

### <a name="em1030-method-m-is-not-generated-because-return-type-t-is-not-supported"></a>EM1030: метод `M` не создается, так как тип возвращаемого значения `T` не поддерживается.

Это **предупреждение о** том, что метод `M` будет игнорироваться (т. е. ничего не будет создано), так как тип возвращаемого значения `T` не поддерживается.

Должно быть более раннее предупреждение, содержащее дополнительные сведения, почему тип `T` не поддерживается.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1031" />

### <a name="em1031-method-m-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1031: метод `M` не создается, так как тип параметра `T` не поддерживается.

Это **предупреждение** , что метод `M` будет игнорироваться (т. е. ничего не будет создано), так как параметр типа `T` не поддерживается.

Должно быть более раннее предупреждение, содержащее дополнительные сведения, почему тип `T` не поддерживается.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1032" />

### <a name="em1032-method-m-has-default-values-for-which-no-wrapper-is-generated"></a>EM1032: метод `M` имеет значения по умолчанию, для которых оболочка не создается.

Это **предупреждение** , что параметры по умолчанию метода `M` не создают никакого дополнительного кода. Наиболее распространенной причиной является то, что существующий метод уже имеет ту же сигнатуру. Например, в .NET может быть:

```csharp
public class MyType {
    public int Increment () { ... }
    public int Increment (int i = 0) { ... }
}
```

В таких случаях будут созданы только два созданных селектора `increment`, оба из которых вызывают моно, но оболочка для более поздних версий не будет существовать.

<a name="EM1033" />

### <a name="em1033-method-m-is-not-generated-because-another-method-exposes-the-operator-with-a-friendly-name"></a>EM1033: метод `M` не создается, так как другой метод предоставляет оператору понятное имя.

Это **предупреждение** , что метод `M` не создается, так как другой метод предоставляет оператору понятное имя. (https://msdn.microsoft.com/library/ms229032(v=vs.110).aspx)

<a name="EM1034" />

### <a name="em1034-extension-method-m-is-not-generated-inside-a-category-because-they-cannot-be-created-on-primitive-type-t-a-normal-static-method-was-generated"></a>EM1034: метод расширения `M` не создается в категории, так как они не могут быть созданы для типа-примитива `T`. Был создан обычный статический метод.

Это **предупреждение о** том, что был найден метод расширения типа примивите (например, `System.Int32`). В цели-C невозможно создать категории для типа-примитива. Вместо этого генератор будет создавать обычный статический метод.

<a name="EM1040" />

### <a name="em1040-property-p-is-not-generated-because-of-parameter-type-t-is-not-supported"></a>EM1040: свойство `P` не создается, так как тип параметра `T` не поддерживается.

Это **предупреждение** , что свойство `P` будет игнорироваться (т. е. ничего не будет создано), так как предоставленный тип `T` не поддерживается.

Должно быть более раннее предупреждение, содержащее дополнительные сведения, почему тип `T` не поддерживается.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1041" />

### <a name="em1041-indexed-properties-on-t-is-not-generated-because-multiple-indexed-properties-are-not-supported"></a>EM1041: индексированные свойства `T` не создаются, так как несколько индексированных свойств не поддерживаются.

Это **предупреждение** , что индексированные свойства на `T` будут игнорироваться (т. е. ничего не будет создано), так как несколько индексированных свойств не поддерживаются.

<a name="EM1050" />

### <a name="em1050-field-f-is-not-generated-because-of-field-type-t-is-not-supported"></a>EM1050: поле `F` не создается, так как тип поля `T` не поддерживается.

Это **предупреждение** , что поле `F` будет игнорироваться (т. е. ничего не будет создано), так как предоставленный тип `T` не поддерживается.

Должно быть более раннее предупреждение, содержащее дополнительные сведения, почему тип `T` не поддерживается.

Примечание. Поддерживаемые функции будут развиваться с помощью новых версий средства.

<a name="EM1051" />

### <a name="em1051-element-e-is-generated-instead-as-f-because-its-name-conflicts-with-an-important-objective-c-selector"></a>EM1051: `E` создается элемент, а не `F`, так как его имя конфликтует с важным селектором цели-c.

Это **предупреждение о** том, что `E` будет создаваться элемент, а не `F`, так как его имя конфликтует с важным селектором "цель-c".

Селекторы в [нсобжектпротокол](https://developer.apple.com/reference/objectivec/1418956-nsobject?language=objc) имеют важное значение в цели-c и должны быть тщательно переопределены.

Примечание. список зарезервированных селекторов будет развиваться с новыми версиями средства.

<a name="EM1052" />

### <a name="em1052-element-e-is-not-generated-its-name-conflicts-with-other-elements-on-the-same-class"></a>EM1052: имя `E` элемента не создано, конфликтует с другими элементами того же класса.

Это **предупреждение** , что элемент `E` не создается, так как его имя конфликтует с другими элементами того же класса.

<a name="EM1053" />

### <a name="em1053-target-e-is-not-supported-for-xamarinios-and-xamarinmac-only-the-framework-option-is-considered-supported-and-should-be-used"></a>EM1053: target `E` не поддерживается для Xamarin. iOS и Xamarin. Mac. Только параметр `framework` считается поддерживаемым и должен использоваться.

Это **предупреждение** , что целевая `E` считается неподдерживаемой для вариантов использования Xamarin. iOS и Xamarin. Mac. 

Использование статических или динамических библиотек внедрения .NET может потребовать дополнительных действий или корректировки, поэтому в большинстве случаев их следует избегать.

Попробуйте удалить параметр `--target` или передать `--target=framework`.
